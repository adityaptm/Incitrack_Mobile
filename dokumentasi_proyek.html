Dokumentasi Lengkap Proyek Incitrack Mobile

Panduan Kode, Arsitektur Sistem, dan Integrasi REST API (Bahan Ajar Pemrograman)

1. Pendahuluan & Arsitektur Sistem

Aplikasi Incitrack Mobile dirancang menggunakan pola arsitektur modern MVVM (Model-View-ViewModel). Arsitektur ini memisahkan logika antarmuka (View), manajemen state (Provider/ViewModel), dan akses jaringan/data (Services & Models). Pola ini sangat disukai di industri karena membuat kode menjadi modular, mudah dibaca, serta ramah untuk pengujian kode (unit testing).

Alur data di dalam sistem bekerja secara reaktif sebagai berikut:

  • View (UI): Berupa Widget Flutter yang mengamati perubahan state dari Provider secara langsung.
  • Provider (State Management): Mengambil data dari Service, memproses data mentah, dan memicu perubahan tampilan UI menggunakan metode notifyListeners().
  • Service (Network API): Berinteraksi langsung dengan backend server menggunakan pustaka http untuk pengiriman data JSON maupun berkas multipart.
  • Model: Memetakan format respon JSON dari REST API Laravel ke dalam objek kelas Dart secara aman.

2. Struktur Direktori Proyek

Struktur folder di dalam direktori lib/ disusun secara modular untuk mendukung pembelajaran tingkat lanjut:

3. Analisis Dependensi (pubspec.yaml)

Dependensi yang dipasang di dalam proyek ini memiliki peran krusial masing-masing:

  • provider: Sebagai jembatan state management untuk mengirimkan data secara horizontal antar layar.
  • flutter_secure_storage: Menyimpan token otentikasi Sanctum secara aman di ruang penyimpanan terenkripsi OS.
  • geolocator: Mengambil data posisi satelit GPS terkini dari sensor internal perangkat secara presisi.
  • flutter_map & latlong2: Peta gratis OpenStreetMap (OSM) tanpa kunci API komersial seperti Google Maps.
  • image_picker: Mengakses fitur kamera perangkat atau galeri untuk mengunggah bukti fisik.

4. Penjelasan Mendalam Kode Sumber

A. Model Laporan (lib/models/laporan_model.dart)

Model ini mengonversi format respons JSON dari Laravel ke dalam kelas objek Dart secara aman dengan proteksi tipe data numerik (koordinat lintang/bujur):

// lib/models/laporan_model.dart
class LaporanModel {
  final int id;
  final int userId;
  final int jalanId;
  final String jenis;
  final String lokasi;
  final double latitude;
  final double longitude;
  final String? penyebab;
  final String? dampak;
  final String? foto;
  final String? video;
  final String status;
  final String createdAt;

  LaporanModel({
    required this.id,
    required this.userId,
    required this.jalanId,
    required this.jenis,
    required this.lokasi,
    required this.latitude,
    required this.longitude,
    this.penyebab,
    this.dampak,
    this.foto,
    this.video,
    required this.status,
    required this.createdAt,
  });

  factory LaporanModel.fromJson(Map<String, dynamic> json) {
    // Menghindari crash konversi double jika API mengembalikan string
    double parseCoordinate(dynamic val) {
      if (val == null) return 0.0;
      if (val is num) return val.toDouble();
      return double.tryParse(val.toString()) ?? 0.0;
    }

    return LaporanModel(
      id: json['id'] ?? 0,
      userId: json['user_id'] ?? 0,
      jalanId: json['jalan_id'] ?? 0,
      jenis: json['jenis_kecelakaan'] ?? json['jenis'] ?? '',
      lokasi: json['lokasi'] ?? '',
      latitude: parseCoordinate(json['latitude'] ?? json['lat']),
      longitude: parseCoordinate(json['longitude'] ?? json['lng']),
      penyebab: json['penyebab'],
      dampak: json['dampak'],
      foto: json['foto_bukti'] ?? json['foto'],
      video: json['video_bukti'] ?? json['video'],
      status: json['status'] ?? 'pending',
      createdAt: json['created_at'] ?? '',
    );
  }
}

B. Integrasi Pengiriman Media Lintas Platform (lib/services/api_service.dart)

Bagian ini menunjukkan bagaimana Flutter menangani upload berkas media (foto/video) secara multipart baik saat berjalan di Platform Web (menggunakan memori bytes) maupun Mobile (menggunakan alamat path berkas lokal):

// lib/services/api_service.dart
static Future<Map<String, dynamic>> kirimLaporan(
  Map<String, String> fields,
  XFile? fotoFile,
  XFile? videoFile,
) async {
  final uri = Uri.parse('${baseUrl}laporan');
  final request = http.MultipartRequest('POST', uri);
  
  final headers = await _getHeaders();
  request.headers.addAll(headers);
  request.headers.remove('Content-Type'); // Biarkan HttpClient membuat pembatas secara otomatis

  request.fields.addAll(fields);

  // Unggah Foto Bukti
  if (fotoFile != null) {
    if (kIsWeb) {
      // Jika berjalan di browser (Flutter Web)
      final bytes = await fotoFile.readAsBytes();
      request.files.add(http.MultipartFile.fromBytes(
        'foto',
        bytes,
        filename: fotoFile.name,
      ));
    } else {
      // Jika berjalan di Android/iOS (Native Mobile)
      request.files.add(await http.MultipartFile.fromPath(
        'foto',
        fotoFile.path,
      ));
    }
  }
  
  final streamedResponse = await request.send();
  final response = await http.Response.fromStream(streamedResponse);
  return jsonDecode(response.body);
}

C. Logika Otomatis Tanggal & Waktu di Provider (lib/providers/laporan_provider.dart)

Agar proses input pengguna minimalis, field tanggal dan waktu secara otomatis dibuat oleh sistem frontend sesaat sebelum payload dikirimkan ke server backend:

// lib/providers/laporan_provider.dart
Future<bool> kirimLaporan({
  required int jalanId,
  required String jenisKecelakaan,
  required String lokasi,
  required double latitude,
  required double longitude,
  String? penyebab,
  String? dampak,
  XFile? fotoFile,
  XFile? videoFile,
}) async {
  _isLoading = true;
  notifyListeners();

  try {
    final now = DateTime.now();
    final Map<String, String> fields = {
      'jalan_id': jalanId.toString(),
      'jenis': jenisKecelakaan,
      'lokasi': lokasi,
      'lat': latitude.toString(),
      'lng': longitude.toString(),
      // Mengirimkan format tanggal YYYY-MM-DD dan waktu HH:mm yang divalidasi server Laravel
      'tanggal': '${now.year}-${now.month.toString().padLeft(2, '0')}-${now.day.toString().padLeft(2, '0')}',
      'waktu': '${now.hour.toString().padLeft(2, '0')}:${now.minute.toString().padLeft(2, '0')}',
    };

    if (penyebab != null && penyebab.isNotEmpty) fields['penyebab'] = penyebab;
    if (dampak != null && dampak.isNotEmpty) fields['dampak'] = dampak;

    final response = await ApiService.kirimLaporan(fields, fotoFile, videoFile);
    return response['success'] == true;
  } catch (e) {
    _errorMessage = e.toString();
    return false;
  } finally {
    _isLoading = false;
    notifyListeners();
  }
}

5. Hubungan Integrasi REST API Laravel

Backend Laravel mendengarkan permintaan API dari Flutter pada rute POST /api/laporan. Pengontrol backend (LaporanApiController.php) memiliki aturan validasi ketat dan penanganan penyimpanan berkas:

  1. Validasi Parameter: Menolak permintaan jika data esensial (seperti jalan_id, tanggal, waktu, jenis, atau koordinat lat / lng) tidak terkirim.
  2. Proses Penanganan Media: Berkas gambar/video yang diunggah akan disaring, diganti namanya menggunakan timestamp acak agar tidak bentrok, dan disimpan di direktori public/uploads/ agar dapat diakses publik.
  3. Status Otomatis: Setiap laporan yang baru saja dikirim oleh pengguna akan disimpan dengan status awal 'pending' sebelum diverifikasi oleh admin web.
